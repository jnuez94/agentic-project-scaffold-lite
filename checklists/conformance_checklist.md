# Conformance Checklist

Use this to determine whether a project is following the working model.

## Records

- [ ] Agent profiles are durable and current.
- [ ] Tasks have owners, statuses, acceptance criteria, and evidence expectations.
- [ ] Messages are durable and findable.
- [ ] Reviews include scope, decision, risks, and blocked claims.
- [ ] Decisions include owner, context, rationale, and implications.
- [ ] Artifacts have owners and usage boundaries.

## Process

- [ ] Agents check coordination records before starting work.
- [ ] Agents announce intent when overlap is possible.
- [ ] Agents request review from the right role.
- [ ] Required changes become tasks.
- [ ] Tasks close only with evidence.
- [ ] Blocked tasks name the blocker and unblocker.
- [ ] Scope changes are recorded as decisions or task updates.

## Safety

- [ ] Sensitive-data policy exists.
- [ ] Coordination records do not store prohibited data.
- [ ] External sharing requires approval.
- [ ] Production launch requires explicit approval.
- [ ] Security or privacy concerns have an owner.
