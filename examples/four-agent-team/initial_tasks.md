# Example Initial Tasks

These starter tasks fit many new projects. Adjust the details to match the project goal.

## TASK-0001: Define Project Goal And Boundaries

Owner: Product Agent

Status: todo

Acceptance criteria:

- project goal is written
- near-term deliverable is written
- non-goals are explicit
- external claims are bounded

Evidence:

- linked project brief or decision record

## TASK-0002: Create Agent Profiles

Owner: Product Agent

Status: todo

Acceptance criteria:

- all active agents have profiles
- responsibilities and decision rights are visible
- operating style includes coordination sync before work

Evidence:

- agent profile files or records

## TASK-0003: Establish Design Baseline

Owner: Design Agent

Status: todo

Acceptance criteria:

- primary user journey is mapped
- key screens or artifacts are identified
- UX risks are documented

Evidence:

- linked flow, mockup, or UX brief

## TASK-0004: Establish Technical Baseline

Owner: Engineering Agent

Status: todo

Acceptance criteria:

- proposed architecture is documented
- core dependencies are named
- implementation risks are documented

Evidence:

- linked technical plan or architecture record

## TASK-0005: Establish Security And Privacy Baseline

Owner: Security And Privacy Agent

Status: todo

Acceptance criteria:

- allowed and prohibited data are documented
- external sharing rules are documented
- release-blocking security concerns are listed

Evidence:

- linked security/privacy baseline

## TASK-0006: Define Release Readiness

Owner: Product Agent

Status: todo

Acceptance criteria:

- release type is defined
- required reviews are listed
- definition of done is written
- blocked claims are written

Evidence:

- release readiness checklist
